sb.datetimeClasses
 


Classes
 ClassDescription
 DateTime An immutable DateTime object. There a multiple ways to create a DateTime.
 DateTimeByDate Creational subclass for DateTime. This creates a DateTime from a Date.
 DateTimeByEpoch Creational subclass for DateTime. Creates a DateTime by the Unix epoch in milliseconds.
 DateTimeByOccurrence Creational subclass for DateTime. Creates a DateTime by an occurence.
 DateTimeByUtcDate Creational subclass for DateTime. This creates a DateTime from a Date using the UTC values.
 DateTimeConstant The DateTime constants class
 DateTimeOccurrenceError Error for creating an invalid DateTimeByOccurrence. The DateTimeByOccurrence constructor will throw this error if the occurence value is 0 or the occurence does not fall with in the month combination. The occurences value cannot be zero. The occurences of the dayOfWeek must fall within the month.
 DST Daylight Savings Time (DST) This currently only supports US DST laws post 2007. This will determine if a TimeZone at a provided date should observer DST.
 Offset The Offset for a TimeZone at a given year, month, day, hour, minute. This does not include a DST offset.
 TimeZone An enumeration of supported Time Zones. This class cannot be not instantiated directly, use the static consts or the getById(id) method. The UsTimeZone class provides easier access to the 7 standard US TimeZones that are most commonly used.
 UsTimeZone A convience class for easy access to the standard US TimeZones Although it is technically incorrect to reference TimeZones by abbreviations, they are commonly well known by them.
